# Football Group Bot (NestJS)

Telegram-бот для организации футбольных игр в группах:
- создание игры только админом;
- запись игроков через кнопки выбора команды;
- автoобновление одного сообщения со списком участников (сообщение закрепляется);
- отправка реквизитов игроку после записи (в личку).
- в ЛС есть кнопки `Оплатил`, `Может не приду`.
- после `Оплатил` у игрока `✅` в общем списке; после `Может не приду` у игрока `❓`.
- команды формируются вручную игроками: 3 кнопки для формата 15, 4 кнопки для формата 20.
- при полном составе бот показывает порядок матчей.

## 1. Установка

```bash
npm install
```

## 2. Настройка

```bash
cp .env.example .env
```

Заполните `BOT_TOKEN`.

## 3. Запуск

```bash
npm run start:dev
```

## 4. Команда создания игры

В группе:

```text
/newgame 2026-03-08 | 20:00 | Арена Юг-2 | 20 | 400 сом | 0555xxxxxx (MBank)
```

После этого бот публикует карточку матча с кнопками:
- кнопки команд (`Красные`, `Синие`, `Белые`, `Черные` для 20 игроков; первые 3 для 15 игроков)
- `❌ Отменить запись`

При каждом действии бот редактирует исходное сообщение и обновляет список игроков.
Создавать игру может только админ группы.
Поддерживаются форматы `15` и `20`.

## Очистка игр в текущем чате

```text
/cleargames
```

## Хранилище

Данные лежат в `storage-data.json` в корне проекта. Это обычный файл, без отдельной БД и без разворачивания сервера хранения.

Для облака задайте `STORAGE_PATH`, например:

```text
STORAGE_PATH=/var/data/storage-data.json
```

## Деплой на Render

1. Залейте проект в GitHub.
2. В Render создайте новый сервис из репозитория (Render Blueprint).
3. Render подхватит [render.yaml](/Users/beastmode/Documents/New project/football-group-bot/render.yaml).
4. В переменных окружения укажите `BOT_TOKEN`.
5. После деплоя бот стартует автоматически, данные будут храниться на постоянном диске `/var/data`.
